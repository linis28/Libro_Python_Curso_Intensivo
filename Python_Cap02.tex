Python difiere ligeramente en sistemas operativos diferentes, por lo que debes tener pendientes

configurar Python correctamente en su sistema.

Hay algunas formas de cambiar los ajustes de configuración predeterminados para VS Code. Algunos cambios se pueden

\subsubsection{Usar tabulaciones y espacios}\label{Tab}

Si utiliza  combinación de tabulaciones y espacios en su código, puede causar problemas que son difíciles de diagnosticar  en
sus programas. Cuando se trabaja en un archivo .py con la extensión Python instalada, VS Code está configurado para insertar cuatro espacios,
cada vez que presione la tecla {\bf{TAB}}. Si estás escribiendo  su propio código y
tienes la extensión Python instalada, probablemente nunca tendrás problemas con tabulaciones y espacios.
Sin embargo, puede ser que su instalación de VS Code no esté configurada correctamente; o que en algún momento debas  trabajar en un archivo que sólo tiene tabulaciones o una mezcla de  tabulaciones y espacios. Si en algún momento sospecha de problema con 
tabulaciones y espacios, consulte la barra de estado en la parte inferior de la ventana de VS Code y haga clic en {\bf{Espacios}}  o  tabulaciones. Aparecerá un menú desplegable que le permitirá cambiar de tabulaciones y a espacios. \subsubsection{Configuración del indicador de longitud de línea}\label{llongitud}

La mayoría de los editores le permiten configurar una señal visual, generalmente es una línea vertical, para mostrar
donde debe terminar sus líneas de código. En la comunidad Python, la convención limite de longitud líneas es 79 caracteres o menos. 

Para configurar esta característica, haga clic en Archivo y selecciona en Preferencias ajustes o configuración. Ahora aparece un cuadro de diálogo, escriba reglas (Rulers). Verás una configuración para Editar reglas;  haga clic en el enlace etiquetado como {\ttfamily{Edit in settings.json}}. Y en el archivo que aparece, agregue la siguiente configuración:\\

{\ttfamily\scriptsize\scriptsize 
	\hspace{-2.1cm}	settings
	
	\hspace{-1.5cm}.json \vspace{-0.7cm}	
	{\color{black} 	
		\begin{verbatim}
			"editor.rulers": [
			80
			]		
		\end{verbatim}
}}


Esto agregará una línea vertical en la ventana de edición en  posición  extremo de 80 caracteres. 
Puedes tener más de una línea vertical; por ejemplo, si quieres
una línea adicional de 120 caracteres, el valor para su configuración sería
[80, 120]. Si no ve las líneas verticales, asegúrese de haber guardado la configuración del
archivo; es posible que también deba salir y volver a abrir VS Code para que se realicen los efectos de cambios en algunos sistemas.

Este proceso también se puede configurar con "{}axTokenizationLineLength{}"  revisar en internet.

\subsection{Atajos de código VS}

Todos los editores e IDE ofrecen formas eficientes de realizar tareas comunes que todos
debe hacer al escribir y mantener el código. Por ejemplo, puedes fácilmente
sangrar una sola línea de código o un bloque completo de código; puedes mover fácilmente un bloque de líneas hacia arriba o hacia
abajo en un archivo.

Hay demasiados atajos para describirlos completamente aquí. Esta sección
comparte solo algunos que probablemente te resulten útiles mientras escribes sus primeros
archivos de Python. Si terminas usando un editor diferente a VS Code, asegúrate
aprenderá a realizar estas mismas tareas de manera eficiente en el editor que haya elegido.

\subsection{Identando Bloques de código con sangría y sin sangría}

Para identar un bloque completo de código, resáltelo y presione {\bf\scriptsize CTRL-]}. 
Para quitar la sangría de un bloque de código, resáltelo y presione {\bf\scriptsize CTRL-]}.

\subsection{Comentar bloques de código}

Para deshabilitar temporalmente un bloque de código, puede resaltar el bloque y
coméntalo para que Python lo ignore. Resalte la sección de código a comenta (ignorar) y presione  {\bf\scriptsize CTRL-/}. 

Las líneas seleccionadas son comentadas con una almohadilla  {\ttfamily {\#}} con sangría al mismo nivel de
línea de código, para indicar que estos no son comentarios regulares. Cuando quieres
para habilitar este bloque de código, resalte el bloque y vuelva a emitir el mismo comando.

\subsection{Mover líneas hacia arriba o hacia abajo} 

A medida que sus programas se vuelven más complejos, es posible que desee mover un
bloque de código hacia arriba o hacia abajo dentro de un archivo. Para hacerlo, resalte el código que desee
para mover y presione: {\bf {}\scriptsize ALT-flecha hacia arriba}. La misma  combinación de teclas con la flecha hacia abajo
moverá el bloque hacia abajo en el archivo.

Si estás moviendo una sola línea hacia arriba o hacia abajo, puedes hacer clic en cualquier parte
esa linea; No es necesario resaltar toda la línea para moverla.

\subsection{Ocultar el Explorador de archivos}

El explorador de archivos integrado en VS Code es realmente conveniente. Sin embargo, puede
puede distraerte cuando escribes código y puede ocupar un espacio valioso en
una pantalla más pequeña. El comando {\bf\scriptsize CTRL-B}, alterna la visibilidad del panel del explorador de archivos.


\subsection{Encontrar atajos adicionales}

Trabajar eficientemente en un entorno de edición requiere práctica, pero también
requiere intención. Cuando esté aprendiendo a trabajar con código, intente notarlas tareas 
que haces repetidamente. Cualquier acción que realice en su editor probablemente tenga un
atajo; Si hace clic en elementos del menú para realizar tareas de edición, busque
los atajos para esas acciones. Si estás cambiando entre su teclado
y el mouse con frecuencia, busque los atajos de navegación que le impiden

Puede ver todos los atajos de teclado en VS Code haciendo clic en el Código y
luego Preferencias y luego eligiendo (métodos abreviado de teclado) Atajos de teclado. Puedes usar la

Recuerde, es mejor centrarse en el código en el que está trabajando y evite dedicar demasiado tiempo a las herramientas que está utilizando.

\section{Propuesta de Mejora de Python (PEPs)}\label{pep8}

% Eric Matthes\\
%   https://es.python.org/      https://www.python.org.ar/

La comunidad de usuarios de Python ha adoptado una guía de estilo que facilita la lectura del código y la consistencia entre programas de distintos usuarios. Esta guía no es de seguimiento obligatorio, pero es altamente recomendable. El documento completo se denomina Propuesta de Mejora de Python (PEPs, Python Enhancement Proposal), es la guía de estilo para Python, \href{https://elpythonista.com/zen-de-python}{\color{vino}código Pythónico}.


En ésta sección se hace una breve exposición de las recomendaciones necesarias al empezar a escribir código Python, y a medida que sea necesario introduciremos las recomendaciones que sean necesarias.

\subsection*{Reglas del Zen de Python}
\addcontentsline{toc}{section}{Reglas del Zen de Python}
% https://www.wikiwand.com/es/Tim_Peters
El Zen de Python es una colección de 20 principios de software que influyen en el diseño del
Lenguaje de Programación Python, de los cuales 19 fueron escritos por Tim Peters\footnote{\scriptsize{Tim Peters es un desarrollador de software estadounidense conocido por la creación del algoritmo de ordenación híbrido Timsort y por sus importantes contribuciones al lenguaje de programación Python y su implementación original CPython. Como usuario de CPython antes de la versión 1.0, formó parte del grupo de los primeros usuarios que contribuyeron al diseño detallado del lenguaje en sus primeras etapas}} en junio de 1999. 

Si queremos escribir código Python que luzca profesional, debemos seguir las recomendaciones del Zen de Python, las  propuesta de mejora de Python, para que nuestros algoritmos sean más elegantes y simples. De aplicarse correctamente el código se convierte {\bf{Código  "Pythónico}}.


\subsection*{Diseñando su estilo de código en Python}  % Styling Your Code    de quien 

Ahora que vas a  empezar  a escribir  programas, es muy buena idea aprender a diseñe su código de manera consistente. Debe hacer lo posible para que su código sea muy fácil leer. Los códigos fácil de leer permiten hacer seguimiento de lo que hace el programas y permite que otros programadores entiendan rápido su código.

Los programadores de Python han acordado una serie de convenciones de estilo, para garantizar que los códigos estén estructurado aproximadamente de la misma manera. Una vez que sabes escribir código Python limpio, deberías poder entender la estructura general
del código Python escrito por cualquier programador, siempre que sigan las mismas pautas.  Si esperas convertirte en un programador profesional en algún momento, debes a seguir estas pautas para desarrollar buenos hábitos.

\subsection{La guía de estilo}\label{PEP}
% https://blog.carreralinux.com.ar/2016/09/convertir-tabulaciones-en-espacios-viceversa/

Una de las  Propuesta de Mejora de Python más antiguos es el PEP 8, que instruye a los programadores de Python sobre cómo diseñar su código. La \href{https://peps.python.org/pep-0008/}{\color{vino}PEP 8} es bastante largo, pero gran parte se relaciona con estructuras de codificación más complejas que las que hemos tratado hasta ahora.  La guía de estilo de Python se escribió entendiendo que el código se lee con más frecuencia de lo que se escribe. Escribirás tu código una vez, y luego comienza a leerlo a medida que comienza a depurar. Cuando agrega funciones a un programa, pasarás más tiempo leyendo tu código. Cuando compartes tu código con otros programadores, ellos también lo leerán. Dada la opción entre escribir código que sea más fácil de escribir, o codificar
un código  que sea más fácil de leer, los programadores de Python casi siempre lo alentarán para escribir código que sea más fácil de leer.

Las siguientes pautas le ayudarán escriba código claro desde el principio.

\subsubsection{Indentación}

La PEP 8 recomienda utilizar cuatro espacios de sangría por nivel, que en Python se conoce como indentación. Usando
cuatro espacios mejoran la legibilidad y dejando espacio para múltiples niveles de indentación en cada línea.


En un documento de procesamiento de textos, la gente suele utilizar tabulaciones en lugar de
espacios para indentación. Esto funciona bien para documentos de procesamiento de textos, pero el
el intérprete de Python se confunde cuando las tabulaciones se mezclan con espacios. 
Cada  editor de texto proporciona una configuración que le permite usar la tecla TAB pero luego convierte
cada tabulación  en un número determinado de espacios. Definitivamente deberías usar tu tecla TAB,
pero asegúrese de que su editor esté configurado para insertar espacios en lugar de tabulaciones en
su documento.

Mezclar tabulaciones y espacios en su archivo puede causar problemas muy
difícil de diagnosticar. Los editores de texto utilizados para desarrollo web a veces no comparten el mismo criterio en cuanto a la cantidad de espacios a los que equivale una tabulación. Por tal motivo, a veces un archivo escrito en un editor XX y que se visualiza de manera correcta en el mismo, otro editor YY lo muestra de forma diferente. Este inconveniente suele ser crítico para programadores Python, lenguaje que requiere de una indentación precisa. Es conveniente convertir tabulaciones en espacios y viceversa de manera que si nos encontramos con archivos que presenten estas características, no representen mayor problema para el trabajo.




\subsubsection{Longitud de la línea}

Muchos programadores de Python recomiendan que cada línea sea menor que
80 caracteres. Históricamente, esta directriz se desarrolló porque la mayoría de las computadoras
sólo podía contener 79 caracteres en una sola línea en una ventana de terminal. Actualmente, las personas
pueden colocar líneas mucho más largas en sus pantallas, pero existen  otras razones
para cumplir con la longitud de línea estándar de 79 caracteres. Los programadores profesionales suelen tener varios archivos abiertos en la mismo pantalla, y usar la longitud de línea estándar les permite ver líneas completas
en dos o tres archivos que están abiertos uno al lado del otro en la pantalla. La PEP 8 también recomienda
que limites todos tus comentarios a 72 caracteres por línea, porque algunas de las herramientas que generan documentación automática para proyectos más grandes agregan caracteres de formato al comienzo de cada
línea comentada.

Las pautas de la PEP 8 para la longitud de la línea no son estrictas, y algunos equipos prefieren un límite de 99 caracteres. 
No te preocupes demasiado por la longitud de la línea en su código,  pero tenga en cuenta que las personas que están trabajando
en colaboración casi siempre siguen las pautas de PEP 8. 
La mayoría de los editores permiten configurar una señal visual, normalmente una línea vertical en la pantalla, que muestra
los esos límites.


En la sección \ref{llongitud}  muestra cómo configurar su editor de texto VS Code para que siempre inserte cuatro
espacios cada vez que presiona la tecla TAB y muestra una guía vertical para ayudarlo a seguir el límite de 79 caracteres.


\subsubsection{Líneas en blanco}

Para agrupar visualmente partes de su programa, utilice líneas en blanco. Deberías usar
líneas en blanco para organizar tus archivos, pero no lo hagas en exceso. 
Siguiendo los ejemplos proporcionados en este libro, deberías lograr un equilibrio adecuado.

Por ejemplo, si tiene cinco líneas de código que crean una lista y luego otras
tres líneas que hacen algo con esa lista, es apropiado colocar una
línea en blanco entre las dos secciones. Sin embargo, no debes colocar tres o cuatro
líneas en blanco entre las dos secciones. Las líneas en blanco no afectarán la forma en que se ejecuta su código, pero sí afectarán la legibilidad de su código. El intérprete de Python usa indentación  horizontal para interpretar el significado de su código, ignorando el espaciado vertical.

\subsubsection*{Otras pautas de estilo}

La PEP 8 tiene muchas recomendaciones de estilo adicionales, pero la mayoría de las
las pautas se refieren a programas más complejos que los tratados hasta ahora. A medida que aprendamos estructuras más complejas de Python, se dará  las directrices de PEP 8.
